# نظام RAG محلي بالعربية

### مساعد أسئلة وأجوبة سيادي يعمل على مستنداتك بالكامل دون اتصال بالإنترنت

**إثبات مفهوم (PoC)** — مبني بما يتوافق مع مبادئ الهيئة السعودية للبيانات والذكاء الاصطناعي (سدايا) للذكاء الاصطناعي التوليدي في الجهات الحكومية.

`ChromaDB` · `bge-m3` · `Ollama` · `Qwen`

</div>

---

## نظرة عامة

نظامٌ يجيب على أسئلتك اعتماداً على مستنداتك الخاصة (سياسات، لوائح، أدلة) باللغة العربية الفصحى، ويعمل **محلياً 100%** — لا تغادر أي بيانات جهاز التشغيل، ولا يتطلب أي اتصال بالإنترنت بعد الإعداد الأولي. صُمِّم النظام ليكون دقيقاً وأميناً على النص المصدر، مع ضوابط تحدّ من «الهلوسة» وتُلزم النموذج بالاستشهاد بالمصدر والصفحات.

| المكوّن | التقنية | الدور |
|---|---|---|
| قاعدة المتجهات | **ChromaDB** | تخزين واسترجاع المقاطع دلالياً |
| نموذج التضمين | **bge-m3** (1024 بُعداً) | دعم عربي قوي — ثابت ولا يُغيَّر بعد الفهرسة |
| نموذج التوليد | **Qwen3.5:9b** (افتراضي، قابل للتبديل) | صياغة الإجابة المقيّدة بالسياق |
| المُشغِّل | **Ollama** | تشغيل النماذج محلياً |
| الواجهة | خادم Python + HTML | محادثة حيّة برفع ملفات وفهرسة فورية |

## المتطلبات

- نظام **macOS** أو **Linux**
- **Python 3.9** أو أحدث
- **[Ollama](https://ollama.com/download)** مُثبَّت ويعمل

> على الماك يمكن تثبيت Ollama عبر `brew install ollama` أو تنزيله من الموقع الرسمي.

## التشغيل السريع

### أول مرة (إعداد كامل بأمر واحد)

من داخل مجلد المشروع في الطرفية:

```bash
bash run_poc.sh
```

يتكفّل هذا الأمر تلقائياً بكل شيء: التحقق من تشغيل Ollama، سحب نموذج التضمين `bge-m3`، فهرسة الملفات في `documents/`، ثم فتح الواجهة على **http://localhost:8765**.

### التشغيلات اللاحقة

```bash
bash run_app.sh      # تشغيل الواجهة فقط (الفهرسة موجودة مسبقاً)
bash run_ingest.sh   # إعادة الفهرسة بعد إضافة أو تعديل ملفات documents/
```

### عبر الطرفية (بدون واجهة)

```bash
.venv/bin/python3 query.py              # حلقة تفاعلية
.venv/bin/python3 query.py "سؤالك هنا"   # سؤال واحد ثم خروج
```

## تبديل نموذج التوليد

اسم نموذج التوليد مُعرَّف في **مكان واحد فقط** — الملف `query.py`، السطر 20:

```python
LLM_MODEL = os.environ.get("RAG_LLM_MODEL", "qwen3.5:9b")
```

للتبديل، اختر إحدى الطريقتين:

**الطريقة الأولى — تعديل الملف:** غيّر `"qwen3.5:9b"` إلى اسم النموذج المطلوب (كما يظهر تماماً في `ollama list`)، ثم:

```bash
ollama pull اسم_النموذج_الجديد
bash run_app.sh
```

**الطريقة الثانية — متغيّر بيئة (دون تعديل الكود):**

```bash
RAG_LLM_MODEL="اسم_النموذج_الجديد" bash run_app.sh
```

> **ملاحظة:** تبديل نموذج التوليد لا يستوجب إعادة الفهرسة. أما نموذج التضمين (`bge-m3`) فثابتٌ إجبارياً — تغييره يفرض إعادة فهرسة كامل المستندات لأن الفهرسة والاستعلام يجب أن يستخدما نفس نموذج التضمين.

## كيف يجيب النظام

يعمل النظام على أربع مراحل مترابطة تضمن الدقة والأمانة على المصدر:

**١. التوجيه الذكي —** يُصنَّف كل مدخل: التحية تُقابَل برد فوري، والسؤال عن النظام نفسه يُعامَل كدردشة، وأي استفسار عن المحتوى يُحال إلى البحث في المستندات.

**٢. الاسترجاع الدلالي —** تُحسب درجة مطابقة مئوية لكل مقطع، مع تعزيز الملف الذي يَرِد اسمه في السؤال والقسم الذي يطابق عنوانه كلمات السؤال، واختيار أقل عدد من الملفات يكفي للإجابة.

**٣. جلب القسم كاملاً —** عند السؤال عن سياسة أو قسم بعينه، تُجلب كل مقاطع ذلك القسم بترتيب القراءة، مع حقن «مراسٍ حتمية» مُستخرجة آلياً (أسماء المبادئ، النطاق، المدد الزمنية) يلتزم النموذج بنسخها حرفياً — بلا نموذج يعني بلا هلوسة.

**٤. صدق المصادر —** يجيب النظام من المستندات ويذكر المصدر والصفحات؛ وإن لم يجد إجابة صرّح بذلك بوضوح، ثم قدّم — عند اللزوم — «اقتراحاً عاماً (ليس من مستنداتك)» مفصولاً بوضوح عن محتوى الوثائق.

## مستويات التفكير

يمكن للمستخدم اختيار مستوى العمق حسب الحاجة، موازنةً بين السرعة والتفصيل والدقة:

| المستوى | المقاطع | السلوك |
|---|---|---|
| 🚀 سريع | 6 | جواب مختصر جداً (جملتان إلى ثلاث) |
| ⚖️ متوازن | 8 | فقرة موجزة وافية |
| 🔬 عميق | 12 | إجابة مفصّلة منظمة بعناوين ونقاط |
| 🎯 دقة قصوى | 12 | مسودة ثم **تدقيق ضد النص الأصلي** — أعلى أمانة، الأبطأ |

من الواجهة: المُحدِّد أعلى حقل الكتابة. من الطرفية: اكتب `عمق سريع` / `عمق متوازن` / `عمق عميق` / `عمق دقيق` أثناء الحوار.

## الامتثال لضوابط سدايا

روعيت في هذا الإصدار مبادئ الهيئة السعودية للبيانات والذكاء الاصطناعي (سدايا) للذكاء الاصطناعي التوليدي في الجهات الحكومية:

**حماية البيانات عند الرفع —** قبل رفع أي ملف عبر الواجهة، يظهر إقرار أمني إلزامي يمنع رفع البيانات المصنّفة (مقيّد، سري، سري للغاية) أو البيانات الشخصية غير المُجهَّلة، ولا تكتمل عملية الرفع إلا بإقرار المستخدم أن الملف يحتوي بيانات **عامة** فقط.

**إخلاء المسؤولية —** بانرٌ دائم أسفل واجهة المحادثة ينبّه إلى أن المخرجات المولّدة قد تكون غير دقيقة أو غير مكتملة، وأنه يجب مراجعتها بشرياً قبل اعتمادها في التقارير الرسمية أو القرارات الحاسمة.

**التشغيل المحلي المعزول —** لا تغادر أي بيانات أو مستندات جهاز التشغيل، فجميع مراحل التضمين والاسترجاع والتوليد تجري محلياً دون اتصال خارجي.

**الأمانة على المصدر —** موجّهات صارمة تُلزم النموذج بنقل الأرقام والمدد الزمنية والمسميات حرفياً من النص الأصلي، وتمنع الاستنتاج أو الإضافة من خارج السياق.

## بنية المشروع

| الملف / المجلد | الوظيفة |
|---|---|
| `documents/` | ملفاتك المصدر (`pdf`, `docx`, `txt`, `md`) |
| `chroma_db/` | قاعدة المتجهات (تُنشأ تلقائياً — غير مُدرجة في المستودع) |
| `ingest.py` | الاستخراج (PyMuPDF للعربية) ← التقطيع بحدود الجمل مع عزل الأقسام وحماية القوائم المرقمة وتتبّع الصفحات ← التضمين ← التخزين، مع تنظيف تلقائي للملفات المحذوفة |
| `query.py` | نواة الاسترجاع والإجابة (تشترك فيها الواجهة والطرفية) — تحتوي كل الموجّهات والإعدادات |
| `app.py` + `ui.html` | واجهة الويب: مراحل حيّة، عدّاد وقت، بثّ الجواب، درجات المطابقة، ورفع الملفات بفهرسة فورية |
| `run_poc.sh` | الإعداد الكامل بأمر واحد |
| `run_ingest.sh` / `run_app.sh` | الفهرسة / تشغيل الواجهة |

## الإعدادات القابلة للضبط

تُضبط جميعها عبر متغيرات بيئة دون تعديل الكود:

| المتغيّر | الافتراضي | الوصف |
|---|---|---|
| `RAG_LLM_MODEL` | `qwen3.5:9b` | نموذج التوليد (بدّله لنموذج أضخم مثل `qwen3:14b`) |
| `RAG_EMBED_MODEL` | `bge-m3` | نموذج التضمين (يستوجب إعادة الفهرسة عند تغييره) |
| `RAG_MAX_DISTANCE` | `0.75` | حدّ الصلة (أقصى مسافة كوساين مقبولة) |
| `RAG_CHUNK_WORDS` | `300` | عدد كلمات المقطع |
| `RAG_OVERLAP_WORDS` | `75` | تداخل المقاطع |
| `RAG_FILE_MARGIN` | `0.10` | هامش قبول ملف إضافي |
| `RAG_NAME_BOOST` | `0.06` | تعزيز الملف المذكور اسمه في السؤال |
| `RAG_HEAD_BOOST` | `0.08` | تعزيز المقطع المطابق لعنوان قسمه |
| `RAG_DEPTH` | — | مستوى التفكير الافتراضي |
