# نظام RAG محلي بالعربية

### مساعد أسئلة وأجوبة سيادي يعمل على مستنداتك بالكامل دون اتصال بالإنترنت

**إثبات مفهوم (PoC)** — مبني بما يتوافق مع مبادئ الهيئة السعودية للبيانات والذكاء الاصطناعي (سدايا) للذكاء الاصطناعي التوليدي في الجهات الحكومية.

`ChromaDB` · `bge-m3` · `Ollama` · `Qwen` · صلاحيات وصول · سجل تدقيق · محادثات متعددة الأدوار

</div>

---

## نظرة عامة

نظامٌ يجيب على الأسئلة اعتماداً على مستندات الجهة (سياسات، لوائح، أدلة) باللغة العربية الفصحى، ويعمل **محلياً 100%** — لا تغادر أي بيانات جهاز التشغيل، ولا يتطلب أي اتصال بالإنترنت بعد الإعداد الأولي. صُمِّم النظام ليكون دقيقاً وأميناً على النص المصدر، مع ضوابط تحدّ من «الهلوسة» وتُلزم النموذج بالاستشهاد بالمصدر والصفحات.

يشمل النظام واجهة محادثة للمستخدم مع سجل محادثات، ولوحة إدارة كاملة، ونظام صلاحيات وصول قائم على المجموعات، وسجل تدقيق، وضوابط أمان مناسبة للبيئات الحكومية.

| المكوّن | التقنية | الدور |
|---|---|---|
| قاعدة المتجهات | **ChromaDB** | تخزين واسترجاع المقاطع دلالياً |
| نموذج التضمين | **bge-m3** (1024 بُعداً) | دعم عربي قوي — ثابت ولا يُغيَّر بعد الفهرسة |
| نموذج التوليد | **Qwen** (افتراضي، قابل للتبديل) | صياغة الإجابة المقيّدة بالسياق |
| نموذج الرؤية/OCR | **qwen2.5vl** (اختياري) | قراءة المستندات الممسوحة والصور |
| المُشغِّل | **Ollama** | تشغيل كل النماذج محلياً |
| الواجهة | خادم Python + HTML | محادثة حيّة، سجل محادثات، رفع ملفات، سمة داكنة/فاتحة |

## أبرز الميزات

- **استرجاع دلالي دقيق** مع اختيار أقل عدد ملفات يكفي، وتعزيز الملف المذكور اسمه في السؤال، وجلب القسم كاملاً عند السؤال عن سياسة بعينها.
- **أمانة على المصدر** — يجيب من المستندات ويذكر المصدر والصفحات، ويصرّح بوضوح إن لم يجد إجابة.
- **محادثات متعددة الأدوار** — سجل محادثات محفوظ في المتصفح مع قائمة جانبية ثابتة، ودعم أسئلة المتابعة اعتماداً على التبادل الأخير.
- **مستويات تفكير** تتحكم بحجم السياق المسترجع وطول الإجابة، وأعدادها قابلة للضبط من لوحة الإدارة لكل مستوى.
- **صلاحيات وصول** قائمة على المجموعات (مستخدمون ← مجموعات مستخدمين ← مجموعات بيانات ← ملفات).
- **لوحة إدارة كاملة** — رفع وفهرسة، إدارة الملفات والمقاطع، منصّة تجربة النموذج، ضبط الإعدادات، وإدارة الصلاحيات.
- **أمان مناسب للجهات الحكومية** — كلمات مرور مجزّأة (pbkdf2)، توكن جلسة للأدمن، سجل تدقيق، وتشغيل محلي معزول.
- **سمة داكنة/فاتحة** مع توهج خلفي خفيف، وزر تبديل يُحفظ اختياره.

## التشغيل السريع

### أول مرة (إعداد كامل بأمر واحد)

من داخل مجلد المشروع في الطرفية:

```bash
bash run_poc.sh
```

يتكفّل هذا الأمر تلقائياً بكل شيء: التحقق من تشغيل Ollama، سحب نموذج التضمين `bge-m3`، فهرسة الملفات في `documents/`، ثم فتح الواجهة على **http://localhost:8765**.

### التشغيلات اللاحقة

```bash
bash run_app.sh      # تشغيل الواجهة فقط (الفهرسة موجودة مسبقاً)
bash run_ingest.sh   # إعادة الفهرسة بعد إضافة أو تعديل ملفات documents/
```

### عبر الطرفية (بدون واجهة)

```bash
.venv/bin/python3 query.py              # حلقة تفاعلية
.venv/bin/python3 query.py "سؤالك هنا"   # سؤال واحد ثم خروج
```

## الاستخدام

### واجهة المستخدم — `http://localhost:8765`

يسجّل الموظف الدخول بحسابه، فتظهر له فقط المستندات المصرّح له بها. الواجهة توفّر:

- **محادثة حيّة** مع بثّ الجواب، وخط سير للمعالجة، ودرجات مطابقة المصادر.
- **سجل محادثات** في قائمة جانبية ثابتة على اليمين (قابلة للإخفاء): محادثة جديدة، إعادة تسمية، حذف، والرجوع لأي محادثة سابقة. يُخزَّن محلياً في المتصفح لكل مستخدم.
- **صندوق إدخال مدمج** فيه قوائم منسدلة لمستوى التفكير ومصدر الإجابة وزر الإرفاق المؤقت.
- **سمة داكنة/فاتحة** عبر زر في أعلى الشاشة.

### لوحة الإدارة — `http://localhost:8765/admin`

تتطلب كلمة مرور المشرف (تُضبط عبر `RAG_ADMIN_PASSWORD`). تشمل:

- **نظرة عامة** بإحصاءات الملفات والمقاطع والإعدادات الحالية.
- **الملفات والمقاطع** — رفع وفهرسة فورية، حذف، إعادة تسمية، واستعراض المقاطع.
- **منصّة تجربة النموذج** — اختبار الأسئلة مع اختيار مجموعة البيانات والمستوى والمصدر (بما فيه «المعرفة العامة» المتاحة للأدمن فقط).
- **الإعدادات** — نموذج التوليد، حد الصلة، عدد المقاطع لكل مستوى تفكير، حجم المقطع والتداخل، وإعدادات OCR.
- **الصلاحيات** — إدارة المستخدمين ومجموعات المستخدمين ومجموعات البيانات.
- **التغذية الراجعة** — استعراض تقييمات المستخدمين.

## مستويات التفكير (حجم السياق)

يمثّل المستوى **حجم السياق المسترجع من المستندات**؛ كلما ارتفع المستوى زاد عدد المقاطع وطالت الإجابة وتفصيلها. الأعداد الافتراضية قابلة للضبط لكل مستوى من لوحة الإدارة.

| المستوى | المقاطع (افتراضي) | السلوك |
|---|---|---|
| 🚀 سريع | 6 | جواب مختصر جداً (جملتان إلى ثلاث) — سياق مختصر |
| ⚖️ متوازن | 8 | فقرة موجزة وافية — سياق متوسط |
| 🔬 عميق | 12 | إجابة مفصّلة منظمة بعناوين ونقاط — سياق موسّع |
| 🎯 دقة قصوى | 12 | مسودة ثم **تدقيق ضد النص الأصلي** — أوسع سياق، الأبطأ |

الفرق بين المستويات ليس في البروبت فقط، بل يشمل: عدد المقاطع المسترجعة (`k`)، وتوسيع الجوار لزيادة اكتمال السياق، وأسلوب الإجابة، ومستوى «دقة قصوى» يضيف تمريرة تحقّق كاملة (استدعاء نموذج إضافي).

من الواجهة يظهر المستوى في المحدِّد المدمج مع شرح مختصر. من الطرفية: اكتب `عمق سريع` / `عمق متوازن` / `عمق عميق` / `عمق دقيق` أثناء الحوار.

## المحادثات متعددة الأدوار

تُخزَّن المحادثات محلياً في متصفح المستخدم (منفصلة لكل مستخدم)، فتبقى بعد تحديث الصفحة ويمكن الرجوع إليها من القائمة الجانبية.

في أسئلة المتابعة يُرسَل **التبادل الأخير فقط** (آخر سؤال وردّه) كسياق إلى النموذج، ويُحوَّل سؤال المتابعة إلى سؤال مستقل قبل البحث لتحسين دقة الاسترجاع. هذا يوازن بين استمرارية الحوار وخفّة الطلب ودقّة الاسترجاع.

## صلاحيات الوصول

نموذج ثلاثي الطبقات يُدار بالكامل من لوحة الإدارة، ويُخزَّن في `access.json`:

```
المستخدم ← مجموعة مستخدمين ← مجموعات بيانات ← ملفات مسموحة
```

- كل مستخدم ينتمي لمجموعة مستخدمين واحدة.
- مجموعة المستخدمين تُمنح وصولاً إلى مجموعة بيانات واحدة أو أكثر (أو «كل البيانات» = وصول كامل).
- مجموعة البيانات = قائمة أسماء ملفات.

عند كل سؤال، يُقيَّد الاسترجاع بالملفات المسموحة للمستخدم فقط — فلا تتسرّب مقاطع من ملفات خارج صلاحيته.

> **دمج مستقبلي مع Microsoft Entra ID (Azure):** التصميم القائم على المجموعات مهيّأ للتكامل مع تسجيل الدخول الموحّد للهيئة. نقطة الاستبدال هي مسار `/user/login` (يُستبدل بتدفق OIDC/OAuth2)، وتبقى طبقة الصلاحيات `user_allowed_sources` كما هي، فقط تُغذّى بمطالبة المجموعة من توكن Azure. توجد علامات توضيحية في الكود عند هذه النقاط.

## الأمان والامتثال لضوابط سدايا

**تجزئة كلمات المرور —** كلمات مرور المستخدمين تُخزَّن مجزّأة بخوارزمية `pbkdf2_sha256` (مكتبة قياسية، دون اعتماد خارجي)، مع ترحيل تلقائي لأي قيمة نصّية صريحة عند أول تشغيل، ومقارنة بزمن ثابت.

**جلسة الأدمن —** لوحة الإدارة تستخدم توكن جلسة مؤقتاً بدل تمرير كلمة المرور في الروابط، وتظهر رسالة تحذير عند استخدام كلمة المرور الافتراضية.

**سجل التدقيق —** يُسجَّل في `audit.jsonl` كل تسجيل دخول (ناجح/فاشل) وكل سؤال (المستخدم، النص، المستوى، المصدر، الوقت) لأغراض الحوكمة والمساءلة.

**حماية البيانات عند الرفع —** قبل رفع أي ملف عبر الواجهة يظهر إقرار أمني إلزامي يمنع رفع البيانات المصنّفة أو الشخصية غير المُجهَّلة، ولا يكتمل الرفع إلا بإقرار المستخدم أن الملف يحتوي بيانات **عامة** فقط.

**إخلاء المسؤولية —** بانرٌ دائم أسفل واجهة المحادثة ينبّه إلى وجوب مراجعة المخرجات بشرياً قبل اعتمادها رسمياً.

**التشغيل المحلي المعزول —** جميع مراحل التضمين والاسترجاع والتوليد تجري محلياً دون اتصال خارجي؛ لا تغادر أي بيانات جهاز التشغيل.

**الأمانة على المصدر —** موجّهات صارمة تُلزم النموذج بنقل الأرقام والمدد الزمنية والمسميات حرفياً من النص الأصلي، وتمنع الإضافة من خارج السياق.

> **ملاحظات ما قبل الإنتاج:** هذا إثبات مفهوم. قبل النشر على سيرفرات الهيئة يُوصى بـ: خادم إنتاجي خلف بروكسي عكسي مع HTTPS بدل خادم التطوير المدمج، وإدارة أسرار خارج المستودع، ونسخ احتياطي لملفات التشغيل (`access.json`, `chroma_db/`, `audit.jsonl`).

## الإعدادات القابلة للضبط

تُضبط عبر متغيرات البيئة، وأغلبها قابل للتعديل الحيّ من لوحة الإدارة أيضاً:

| المتغيّر | الافتراضي | الوصف |
|---|---|---|
| `RAG_LLM_MODEL` | `qwen3.5:9b` | نموذج التوليد (بدّله لنموذج أكبر مثل `qwen3:14b`) |
| `RAG_EMBED_MODEL` | `bge-m3` | نموذج التضمين (يستوجب إعادة الفهرسة عند تغييره) |
| `RAG_MAX_DISTANCE` | `0.75` | حدّ الصلة (أقصى مسافة كوساين مقبولة) |
| `RAG_CHUNK_WORDS` | `300` | عدد كلمات المقطع |
| `RAG_OVERLAP_WORDS` | `75` | تداخل المقاطع |
| `RAG_FILE_MARGIN` | `0.10` | هامش قبول ملف إضافي |
| `RAG_NAME_BOOST` | `0.06` | تعزيز الملف المذكور اسمه في السؤال |
| `RAG_HEAD_BOOST` | `0.08` | تعزيز المقطع المطابق لعنوان قسمه |
| `RAG_DEPTH` | `balanced` | مستوى التفكير الافتراضي |
| `RAG_ADMIN_PASSWORD` | `admin` | كلمة مرور لوحة الإدارة (**غيّرها قبل أي تشغيل غير محلي**) |
| `RAG_OCR` | `1` | تفعيل OCR للمستندات الممسوحة والصور |
| `RAG_OCR_MODEL` | `qwen2.5vl:3b` | نموذج الرؤية للـ OCR (للجهاز القوي `qwen2.5vl:32b`) |

> عدد المقاطع لكل مستوى تفكير يُضبط من لوحة الإدارة (تبويب الإعدادات) ويُطبَّق عالمياً فوراً.

## تبديل نموذج التوليد

اسم نموذج التوليد مُعرَّف في `query.py` (`LLM_MODEL`). للتبديل:

```bash
ollama pull اسم_النموذج_الجديد
RAG_LLM_MODEL="اسم_النموذج_الجديد" bash run_app.sh
```

أو غيّره حيّاً من لوحة الإدارة. تبديل نموذج التوليد لا يستوجب إعادة الفهرسة. أما نموذج التضمين (`bge-m3`) فثابتٌ إجبارياً — تغييره يفرض إعادة فهرسة كامل المستندات لأن الفهرسة والاستعلام يجب أن يستخدما نفس نموذج التضمين.

## بنية المشروع

| الملف / المجلد | الوظيفة |
|---|---|
| `documents/` | ملفات المصدر (`pdf`, `docx`, `txt`, `md`, صور) |
| `chroma_db/` | قاعدة المتجهات (تُنشأ تلقائياً — غير مُدرجة في المستودع) |
| `ingest.py` | الاستخراج (PyMuPDF + OCR للعربية) ← التقطيع بحدود الجمل مع عزل الأقسام وتتبّع الصفحات ← التضمين ← التخزين |
| `query.py` | نواة الاسترجاع والإجابة (تشترك فيها الواجهة والطرفية) — الموجّهات والإعدادات والتوجيه الذكي |
| `app.py` | خادم الويب: المسارات، المصادقة، الصلاحيات، سجل التدقيق، ولوحة الإدارة |
| `ui.html` | واجهة المستخدم: المحادثة، سجل المحادثات، السمة، الإرفاق |
| `admin.html` | لوحة الإدارة |
| `access.json` | المستخدمون والصلاحيات (كلمات مرور مجزّأة — غير مُدرج في المستودع) |
| `audit.jsonl` | سجل التدقيق (غير مُدرج في المستودع) |
| `run_poc.sh` | الإعداد الكامل بأمر واحد |
| `run_ingest.sh` / `run_app.sh` | الفهرسة / تشغيل الواجهة |

> **ملفات لا تُرفع للمستودع** (في `.gitignore`): `access.json`، `chroma_db/`، `audit.jsonl`، `feedback.jsonl`، البيئة الافتراضية، والمرفقات المؤقتة.
